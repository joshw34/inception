ssl/tls

  • TLS = modern version of SSL (SSL is deprecated/insecure)

  • Private key = key.pem (secret, stays on server)

  • Public key = included in cert.pem (public, sent to everyone)

  • Regular HTTP is unencrypted - data sent in plain text (can be intercepted)

  • HTTPS addresses this via public key encryption (asymmetric) + session keys (symmetric)

  • Something encrypted with the public key can only be decrypted with the private key

  • Encryption process:

    • Site sends cert to user (contains public key)

    • User generates random session key and encrypts it using our public key

    • We decrypt this session key using our private key

    • This session key is now known only by us and the user

    • All further communication uses this session key (fast symmetric encryption)

    • Any intercepted data is useless without the session key

  • Why self-signed certificates show warnings:

    • Encryption still works perfectly

    • Warning is about TRUST (can't verify you own the domain)

    • CA-signed certificates are trusted because a Certificate Authority vouches for you

  • Inception project:

    • NGINX terminates SSL/TLS (decrypts HTTPS traffic)

    • Internal container communication (NGINX → WordPress → MariaDB) is unencrypted.

      • This is ok because it's on a private Docker network

      • No external access to internal traffic

    • Only port 443 (HTTPS) is exposed externally
